Security & Compliance

HIPAA-conscious architecture

✓ All Systems Secure
🔐
Role-Based Access Control
PTs see only their own patients. Patients see only their own data. Role claims are server-set in JWT and cannot be self-modified.
JWT claims verified server-side
RLS on every PHI table
Zero cross-patient data leakage
🛡
PHI Encryption
Sensitive data encrypted with AES-256 via pgcrypto at rest. TLS 1.3 in transit. NO free-text messaging eliminates largest HIPAA risk vector.
AES-256 at rest
TLS 1.3 in transit
No PHI in messaging (none exists)
📋
Immutable Audit Logs
Every action writes to INSERT-only audit_logs. No UPDATE or DELETE ever permitted. Partitioned by month for scale.
INSERT-only table
IP address captured per action
Monthly partitions for scale
⚡
No Messaging Architecture
Zero free-text communication between patients and therapists. All communication is pre-structured check-ins and system-generated notifications only.
No messaging = no HIPAA chat risk
Pre-structured responses only
System-generated alerts only
🚫
Rate Limiting
Upstash Redis sliding window at Edge Middleware. Auth: 10 req/min. API: 60 req/min. Auto lockout after repeated failures.
Edge middleware enforcement
Per-IP and per-user limits
Brute force protection
✅
Session Security
JWT tokens expire in 1 hour. Refresh tokens rotate on every use. PTs must complete TOTP MFA. Idle sessions expire at 60 minutes.
1hr JWT expiry + refresh rotation
TOTP MFA enforced for all PTs
PKCE flow for mobile
